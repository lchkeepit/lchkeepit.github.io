Splay

splay是一种独特的二叉排序树，通过rotate和splay操作，将需要处理的节点旋转至跟或是根的儿子处，在对其进行操作。也就是说splay天生就有一种类似缓存的能力。splay不能保证单次操作的复杂度O（logn）但是它能保证它的均摊复杂度o(logn).

rotate操作

void rotate(int r,int kind)
{
    int y = pre[r];
    pushdown(y);
    pushdown(r);
    ch[y][!kind] = ch[r][kind];
    pre[ch[r][kind]] = y;
    if (pre[y])
        ch[pre[y]][ch[pre[y]][1] == y] = r;
    ch[r][kind] = y;
    pre[r] = pre[y];
    pre[y] = r;
    pushup(y);
}

rotate分为左旋和右旋，这里用kind巧妙的减少了代码的冗余

Splay操作

void splay(int r,int goal)
{
    pushdown(r);
    while (pre[r] != goal)
    {
        if (pre[pre[r]] == goal)
        {
            pushdown(pre[r]);
            pushdown(r);
            int kind = ch[pre[r]][0] == r;
            rotate(r,kind);
        }
        else
        {
            int y = pre[r];
            pushdown(pre[y]);
            pushdown(y);
            pushdown(r);
            int kind = ch[pre[y]][0] == y;
            if (ch[y][kind] == r)
            {
                rotate(r,!kind);
                rotate(r,kind);
            }
            else
            {
                rotate(y,kind);
                rotate(r,kind);
            }

        }
    }
        pushup(r);
    if (goal == 0) root = r;
}

大家注意到这里pushdown的非常频繁，这是因为splay能实现一个lazy标记的翻转操作，翻转之后左右儿子需要交换才行

另外在翻转操作传递lazy标记的时候

void pushrev(int r)
{
    if (r == 0) return ;
    swap(ch[r][0],ch[r][1]);
    rev[r] ^= 1;
}

一定要先把当前节点翻转，才打标记，容易遗忘，类比线段树。
